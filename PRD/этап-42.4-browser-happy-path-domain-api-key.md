# PRD: Этап 42.4. Browser happy-path для `domain + api_key`

## Контекст

Этапы `42.1`–`42.3` уже подготовили:
- рабочий Playwright stack в default pytest suite;
- live localhost HTTP harness;
- deterministic upstream mocks для auth flow.

Следующий шаг — закрыть полноценный browser happy-path для API-key режима
без зависимости от внешнего Vetmanager.

## Цель 42.4

Добавить browser regression test, который проходит путь:
- регистрация account;
- вход/наличие active session;
- сохранение Vetmanager integration через `domain + api_key`;
- выпуск service bearer token;
- `mcp.call_tool(...)` с этим bearer token.

## Входит

- Browser test через Playwright.
- Использование live localhost harness.
- Использование deterministic upstream mock для `domain + api_key`.
- Проверка, что bearer token действительно работает для MCP runtime.

## Не входит

- Login/password browser happy-path.
- Cleanup helper и cleanup regression.
- Optional real browser mode.

## Декомпозиция

### 42.4.1 Артефакты

### 42.4.2 Red test

### 42.4.3 Green

## Критерии готовности

- Browser test проходит без внешнего Vetmanager.
- Happy-path заканчивается успешным `mcp.call_tool(...)`.
- Проверяется, что upstream client endpoint реально вызван bearer-driven runtime.

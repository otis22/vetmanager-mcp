# PRD Этап 14: Универсальная сортировка и фильтрация list GET

## Цель
Добавить во все list `get_*` инструменты унифицированные параметры `sort` и `filter` в формате Vetmanager API, чтобы агент мог выполнять сложные выборки без ручного добавления узких параметров в каждый метод.

## Контекст
- Сейчас list-инструменты в основном поддерживают `limit/offset` и несколько точечных фильтров (`name`, `client_id`, `date` и т.д.).
- В API Vetmanager доступны универсальные `sort` и `filter` с операторами (`=`, `!=`, `<>`, `<`, `<=`, `>`, `>=`, `in`, `not in`, `like`).
- Нужна единообразная реализация во всех list GET без расхождения между модулями `tools/*.py`.

## Контракт
- Для всех list `get_*` инструментов:
  - `sort` — optional массив объектов вида `{"property": "...", "direction": "ASC|DESC"}`.
  - `filter` — optional массив объектов вида `{"property": "...", "value": ..., "operator": "..."}`.
- Перед отправкой в query параметры `sort` и `filter` сериализуются в JSON-строку.
- Существующие точечные параметры фильтрации сохраняются (для простых кейсов), но дополняются универсальными `sort/filter`.

# PRD: Этап 42.5. Browser happy-path для `login/password -> user token`

## Контекст

После `42.4` browser suite уже покрывает happy-path для `domain + api_key`.
Остаётся закрыть второй поддерживаемый auth mode, в котором web UI:
- принимает `domain`, `login`, `password`;
- обменивает их на user token через `token_auth.php`;
- хранит только user token;
- выпускает service bearer token для MCP runtime.

## Цель 42.5

Добавить browser regression test на полный happy-path для user-token режима:
- регистрация account;
- переключение wizard на `login/password`;
- token exchange и сохранение integration;
- выпуск service bearer token;
- `mcp.call_tool(...)` через bearer runtime.

## Входит

- Playwright browser test.
- Live localhost harness.
- Deterministic user-token upstream mocks.
- Проверка, что MCP runtime реально использует выданный user token.

## Не входит

- UI secrecy assertions как отдельный этап.
- Cleanup helper и cleanup regression.
- Optional real browser mode.

### 42.5.3 Green

## Критерии готовности

- Browser user-token flow проходит без внешнего Vetmanager.
- Happy-path заканчивается успешным bearer-driven `mcp.call_tool(...)`.
- Upstream `/token_auth.php` и `/rest/api/user` реально задействованы в тесте.

# PRD: Этап 42.6. Browser assertions на UI-контракт и отсутствие утечек секретов

## Контекст

Этапы `42.4` и `42.5` уже закрывают browser happy-path для обоих auth modes.
Теперь нужно зафиксировать на browser-уровне не только успех сценария, но и
контракт UI после submit:
- какие поля реально присутствуют в DOM;
- что секреты не всплывают обратно в HTML после сохранения integration.

## Цель 42.6

Усилить browser tests явными assertions на:
- UI contract wizard/форм;
- отсутствие утечек `api_key`, `vm_password`, `user_token` после submit.

## Входит

- Добавление assertions в существующие browser happy-path tests.
- Проверка DOM-контракта по полям auth wizard.
- Проверка отсутствия секретов в итоговом page content.

## Не входит

- Cleanup helper.
- Optional real browser mode.

## Критерии готовности

- Browser tests явно проверяют DOM-контракт auth wizard.
- Browser tests явно подтверждают, что секреты не возвращаются в HTML после submit.

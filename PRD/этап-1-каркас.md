# PRD Этап 1: Базовый каркас сервиса

## Цель
Создать минимальный работающий каркас MCP-сервера с клиентом Vetmanager API, поддержкой мультитенантности (domain/api_key на каждый запрос), обработкой ошибок и логированием.
Дополнительно: весь workflow проекта должен быть docker-only, без зависимости от Python на хосте.

## Задачи и декомпозиция

### 1.1 Инициализация проекта
- Создать `pyproject.toml` с зависимостями: `mcp[cli]`, `httpx`, `pydantic`
- Создать структуру: `server.py`, `vetmanager_client.py`, `exceptions.py`, `tools/__init__.py`
- Добавить Dockerfile и базовую конфигурацию для запуска через `docker compose`
- Ограничение: ≤ 30 строк на файл инициализации

### 1.2 Мультитенантная конфигурация
- `domain` и `api_key` принимаются как параметры каждого MCP-инструмента
- Никакой глобальной привязки к одному домену/ключу
- Ограничение: ≤ 30 строк

### 1.3 API-клиент с динамическим URL
- Класс `VetmanagerClient(domain, api_key)`
- Метод `_resolve_host()` → GET `https://billing-api.vetmanager.cloud/host/{domain}`
- Async методы: `get`, `post`, `put`, `delete`
- Кэш URL в памяти на уровне экземпляра
- Ограничение: ≤ 120 строк

### 1.4 Исключения и логирование
- `exceptions.py`: `VetmanagerError`, `AuthError`, `NotFoundError`, `TimeoutError`
- Логирование в stderr через stdlib `logging`, уровень из env `LOG_LEVEL`
- Ограничение: ≤ 40 строк

### 1.5 Docker-only исполнение и права
- Любые команды Python/CLI выполняются только в контейнере (`docker compose run/exec`)
- Контейнер запускается от non-root пользователя с UID/GID хоста
- Проверить, что файлы, измененные из контейнера, остаются доступными пользователю хоста
- Ограничение: ≤ 40 строк

### 1.6 Тест мультитенантности (unit)
- Проверить что два разных `VetmanagerClient(domain1, ...) / VetmanagerClient(domain2, ...)` независимы
- Ограничение: ≤ 40 строк

## Критерии готовности
- `uv run server.py` запускается без ошибок
- Запуск сервиса, CLI и тестов выполняется через `docker compose`, без Python на хосте
- `VetmanagerClient` принимает domain/api_key в конструкторе
- Unit-тест на независимость клиентов проходит
- Никаких hardcoded доменов/ключей в коде
- Нет проблем с правами на файлы после операций из контейнера

# PRD Этап 15.4: MCP Prompts на headers-only контракте

## Цель
Привести `prompts.py` в соответствие с текущей архитектурой проекта:
runtime credentials больше не передаются в prompt-функции и не упоминаются
как аргументы tool calls. Все credentials приходят только через HTTP headers
`X-VM-Domain` / `X-VM-Api-Key`.

## Проблема
- После этапов 11–12 tools и сервер уже работают по headers-only контракту.
- `prompts.py` всё ещё принимает `domain` и `api_key` в сигнатурах.
- Тексты prompts подсказывают LLM использовать `domain` и `api_key` в вызовах,
  что противоречит текущему runtime-контракту и может путать MCP-клиенты.
- Отдельной проверки на этот контракт в тестах нет.

## Границы задачи
- Обновить только `prompts.py`, тесты на prompts, README и AssumptionLog.
- Не менять transport, tools/list и не трогать реализацию инструментов,
  кроме случаев, когда это нужно для точности формулировок в prompts.

## Декомпозиция

### 15.4.1 Сигнатуры prompts
- Убрать `domain` / `api_key` из сигнатур всех 20 prompt-функций.
- Оставить только бизнес-параметры сценариев.

### 15.4.2 Тексты prompts
- Удалить инструкции вида `Use domain=...` / `api_key=...`.
- Добавить единое headers-only правило:
  credentials уже доступны из request headers, их не нужно спрашивать
  у пользователя и нельзя передавать как tool arguments.

### 15.4.3 Точность инструкций
- Сверить упоминаемые tool names и параметры с текущими `tools/*.py`.
- Убрать подсказки про несуществующие аргументы tool calls.

### 15.4.4 Проверка
- Добавить тест на контракт prompts:
  - нет аргументов `domain` / `api_key`;
  - нет старых текстовых подсказок про передачу credentials;
  - есть headers-only инструкция.

### 15.4.5 Документация
- Обновить раздел `MCP Prompts` в `README.md`.
- Зафиксировать решение в `AssumptionLog.md`.

## Критерии готовности
- Все prompt-функции принимают только бизнес-параметры.
- В prompts нет инструкций передавать credentials в tool calls.
- Есть тест, защищающий от регрессии.
- `README.md`, `Roadmap.md`, `AssumptionLog.md` синхронизированы.

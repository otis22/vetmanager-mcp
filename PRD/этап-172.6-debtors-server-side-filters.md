# Этап 172.6. `get_debtors` server-side filters

## Контекст

Production feedback `#5`: `get_debtors` падает на большой клиентской базе до полезного результата. Текущий tool вызывает `paginate_all("/rest/api/client", status=ACTIVE)`, затем фильтрует `balance < 0` в Python. На больших базах срабатывает `max_rows` до того, как пользователь получает должников.

## Проверенные факты

- `artifacts/api_entity_reference-ru.md` подтверждает поля `client.balance` и `client.last_visit_date`.
- Report AI research 2026-06-15 проверял прямой `/rest/api/client` filter `balance < 0` на `devtr6`; он возвращал ожидаемых debtors.
- `get_inactive_clients` уже использует server-side `last_visit_date` filters и sort для клиентских окон.
- `paginate_all` имеет default `max_rows=10_000` и правильно падает на слишком широких выборках; проблема в том, что `get_debtors` сначала выбирает всех ACTIVE.

## Цель

Сделать `get_debtors` bounded и пригодным для больших баз: фильтровать debtors на стороне Vetmanager API и возвращать страницу результатов.

## Scope

### In scope

- Заменить broad `paginate_all` на single page `/rest/api/client` request.
- Всегда отправлять filters:
  - `status = ACTIVE`;
  - `balance < 0`.
- Добавить optional filters:
  - `last_visit_date_from`;
  - `last_visit_date_to`.
- Использовать `limit`/`offset` как реальную страницу debtors.
- Использовать deterministic sort `id ASC`, чтобы `limit/offset` страницы не дублировались и не пропускали записи.
- Вернуть metadata: `returned_count`, `total_count`, `limit`, `offset`, `applied_filters`, `sort`, `server_side_balance_filter`.
- Сохранить совместимые поля `success`, `debtors_count`, `debtors`, `total_active_clients_checked`.
- Новая семантика:
  - `debtors_count` = `returned_count` текущей страницы;
  - `total_count` = upstream `totalCount` для server-side filtered debtors;
  - `total_active_clients_checked` = compatibility alias для `total_count`, больше не означает full scan всех ACTIVE clients.
- Локально оставить defensive `balance < 0` post-filter на полученной странице, если upstream когда-либо вернёт лишние строки.

### Out of scope

- Автоматический full export всех должников.
- Report AI debtors reports.
- Изменение semantics знака долга; текущий production feedback говорит про negative balance.

## Acceptance

- `get_debtors()` отправляет `/rest/api/client` с `status=ACTIVE` и `balance < 0`, не вызывает `paginate_all`.
- `limit`, `offset` и deterministic `sort=[{"property":"id","direction":"ASC"}]` попадают в API request как страница результата.
- `last_visit_date_from/to` добавляют server-side date window.
- Ответ содержит `applied_filters`, `total_count`, `returned_count`, `total_active_clients_checked`, `server_side_balance_filter=true`.
- Targeted Docker tests pass.

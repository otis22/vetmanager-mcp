# PRD Этап 8: Защита от ошибок — лимиты на массовые операции

## Цель
Предотвратить случайные массовые изменения или чрезмерные выборки данных,
вызванные ошибочными промптами пользователя.

## Реализация

### validators.py
Функция `validate_list_params(limit, offset)` — вызывается в начале каждого
get_* list-инструмента. При нарушении — `ValueError` с понятным сообщением.

Лимиты:
- `limit`: 1 ≤ limit ≤ 100
- `offset`: 0 ≤ offset ≤ 10 000

### create_payment
Дополнительная валидация `amount`: 0 < amount ≤ 1 000 000

## Тесты
`tests/test_validators.py` — 6 сценариев:
- limit=1 → ok
- limit=100 → ok
- limit=0 → ValueError
- limit=101 → ValueError
- offset=0 → ok
- offset=10000 → ok
- offset=10001 → ValueError
- amount=0.01 → ok
- amount=1_000_000 → ok
- amount=0 → ValueError
- amount=1_000_001 → ValueError

## Критерии готовности
- Все get_* list-инструменты используют validate_list_params
- Тесты проходят

## Пометка по Variant A (плановая)
- Для рабочего контура источник `domain`/`api_key` — клиентский `mcp.json`
  (`url` + `headers`), а не проектный runtime env.
- Ограничения этапа 8 применяются независимо от источника credentials
  и не меняют правило, что `TEST_*` используются только в real e2e тестах.

# PRD: Этап 27.1.x. Legacy test/runtime refactoring под новый credential-контракт

## Цель

Убрать рассинхрон между актуальным runtime-контрактом bearer-only сервиса и
старым test/support кодом, который всё ещё создаёт `VetmanagerAuthContext`
через устаревший аргумент `api_key` и вручную подменяет внутреннее состояние
`VetmanagerClient`.

## Контекст

- Актуальный runtime слой уже нормализован вокруг:
  - `Authorization: Bearer <service_token>`;
  - `RuntimeCredentials`;
  - `VetmanagerAuthContext(credential=...)`.
- В коде при этом остались legacy tests/helpers, которые:
  - напрямую создают `VetmanagerAuthContext(api_key=...)`;
  - дублируют setup `VetmanagerClient`;
  - частично обходят текущий runtime boundary через ручное присваивание полей.
- Первый подтверждённый сбой полного suite:
  `tests/test_client_multitenancy.py` падает на `TypeError` из-за старой
  сигнатуры `VetmanagerAuthContext`.

## Что входит в текущий срез

- Зафиксировать аудит legacy helper-слоя.
- Вынести общий test factory для:
  - bearer headers;
  - `VetmanagerAuthContext`;
  - `RuntimeCredentials`;
  - преднастроенного `VetmanagerClient`.
- Перевести `tests/test_client_multitenancy.py` на новый helper.

## Что не входит в текущий срез

- Полная миграция `tests/test_e2e_mock.py`.
- Чистка всех остальных старых тестов в репозитории.
- Удаление backward-compatible alias `api_key` из runtime-классов.

## Принципы рефакторинга

- Публичный runtime-контракт не меняется.
- `api_key` остаётся допустимым названием только для connection payload
  режима `domain_api_key`.
- Внутри runtime/test helpers каноническим именем считается `credential`.
- Тесты должны пользоваться общими factory/helper функциями вместо ручного
  копирования внутреннего состояния клиента.

## Декомпозиция

### 27.1.1 Аудит
- Найти legacy tests/helpers, завязанные на `api_key=` в `VetmanagerAuthContext`.
- Зафиксировать приоритет миграции: `test_client_multitenancy`, затем `test_e2e_mock`.

### 27.1.2 Общие test factories
- Добавить helper-модуль для сборки:
  - `VetmanagerAuthContext`;
  - `RuntimeCredentials`;
  - `VetmanagerClient` с уже подменённым runtime resolution.
- Снизить дублирование и сделать future migration дешевле.

### 27.1.3 Первая миграция
- Перевести `tests/test_client_multitenancy.py` на новые helpers.
- Убедиться, что file-level intent тестов не меняется:
  transport, cache, pacing, host resolution.

### 27.1.4 Следующая волна
- Аналогично перевести `tests/test_e2e_mock.py` и связанные helper paths.

### 27.1.5 Verification
- Прогнать таргетные тесты.
- После миграции всех legacy tests прогнать полный suite.

## Критерии готовности для текущего среза

- В репозитории есть отдельный PRD под refactoring legacy runtime helpers.
- Появился общий test helper/factory модуль.
- `tests/test_client_multitenancy.py` больше не использует
  `VetmanagerAuthContext(api_key=...)`.
- Таргетный срез тестов по `client_multitenancy` проходит.

# Этап 129.6. Rate-limit backend shutdown and cleanup

## Контекст

`reset_rate_limit_backend()` просто зануляет singleton backend. Для Redis backend это оставляет
async client lifecycle на GC вместо явного `aclose()`.

## Цель

Добавить явный shutdown/close path для backend instance и использовать его в reset helper.

## Scope

1. Добавить `close()` в backend contract.
2. Добавить `shutdown_rate_limit_backend()` для явного async cleanup.
3. Обновить sync `reset_rate_limit_backend()` так, чтобы он тоже запускал cleanup path.
4. Добавить regression tests на `aclose()` для Redis-backed instance.

# PRD Этап 10: Variant A (mcp.json `url` + `headers`) — только документация и задачи

## Цель
Скорректировать управленческие артефакты проекта под целевую схему подключения:

- клиент (Cursor) подключается к MCP по `url` (`http://localhost:8000/mcp`);
- `domain` и `api_key` передаются из пользовательского `mcp.json` в `headers`;
- runtime credentials не хранятся в репозитории проекта;
- тестовые credentials существуют только для e2e real tests.

Важно: в рамках этапа 10 **код сервиса не меняется**. Обновляются только Roadmap/PRD-формулировки и постановка следующих задач.

## Контекст
Текущее состояние документов частично смешивает два подхода:

1) мультитенантные параметры на каждый запрос;
2) fallback из проектного env.

Целевой вариант для дальнейшей реализации — **Variant A**:
пользовательские credentials живут в клиентском конфиге и приходят в каждом запросе,
проект хранит только тестовый контур (`TEST_DOMAIN`/`TEST_API_KEY`).

# PRD Этап 3: Docker Compose локальное окружение

## Цель
Настроить полностью docker-only workflow: запуск сервиса, тестов и любых CLI-команд исключительно через `docker compose`. Контейнер работает от non-root пользователя с UID/GID хоста.

## Задачи

### 3.1 docker-compose.yml
- Сервис `mcp` для основного сервера
- Сервис `test` для прогона тестов (profile: test)
- Передача UID/GID хоста через переменные
- Ограничение: ≤ 50 строк

### 3.2 .env.example
- Шаблон без реальных ключей
- Объяснение назначения каждой переменной
- Ограничение: ≤ 15 строк

### 3.3-3.6 Права и CLI
- UID/GID из окружения хоста
- Non-root user в Dockerfile (уже готово в Этапе 1)
- Все команды через `docker compose run --rm test`

## Критерии готовности
- `docker compose build` проходит
- `docker compose run --rm test` запускает тесты
- Созданные файлы принадлежат пользователю хоста

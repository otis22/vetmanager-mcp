# PRD: Этап 42.2. Live HTTP test harness для browser tests

## Контекст

Этап 42.1 добавил browser stack в стандартный `pytest` и test container:
Playwright и Chromium теперь доступны в default suite.

Следующий шаг нужен для реальных browser happy-path сценариев: browser tests
должны работать не через `httpx.ASGITransport`, а через живой HTTP endpoint на
`localhost`, чтобы браузер проходил полноценный сетевой путь:
- real navigation;
- cookies;
- redirects;
- формы и submit;
- CSP/security headers в реальном HTTP ответе.

## Цель 42.2

Поднять в обычном `pytest` live HTTP harness для browser tests и встроить его в
общий запуск тестов, не ломая существующий async/unit/mock suite.

## Входит в задачу

- PRD и обновление workplan.
- Минимальный browser regression test, который требует именно live HTTP URL.
- Общие pytest fixtures для:
  - временной web-БД;
  - создания ASGI app;
  - запуска live HTTP server на свободном localhost-порту;
  - корректного shutdown и очистки состояния.
- Интеграция harness в обычный `docker compose --profile test run --rm test`.

## Не входит

- Полный happy-path account -> integration -> bearer -> MCP call.
- Upstream mocks Vetmanager для двух auth flow.
- Cleanup test accounts и regression tests на cleanup.

## Решения

### Live server

- Использовать реальный HTTP server поверх текущего ASGI app `mcp.http_app(...)`.
- Server должен слушать `127.0.0.1` на динамически выбранном свободном порту.
- Startup/shutdown должны быть управляемы из pytest fixture без внешних ручных
  команд.

### Изоляция

- Каждый browser/live test получает отдельную SQLite БД во временном каталоге.
- Перед запуском app нужно сбросить process-level storage/web security state.
- После теста server останавливается, engine dispose выполняется, состояние
  storage/security сбрасывается повторно.

### Минимальная проверка

- Browser test должен открыть live URL через `page.goto(...)`, а не `data:` URL.
- Проверка должна убедиться, что через live HTTP реально доступна web-страница
  проекта, например `/register` или `/`.

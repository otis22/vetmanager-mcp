## Цель

Добавить два агрегирующих MCP-инструмента:
- `get_client_profile` — возвращает полный профиль клиента: данные, счета с номенклатурой, визиты, следующий приём.
- `get_pet_profile` — возвращает полный профиль питомца: данные, медкарты с диагнозами, история вакцинаций.

Инструменты позволяют AI-агенту за один вызов получить всю клинически и финансово значимую информацию.

## Исследование API (проведено 2026-02-28)

### Фильтрация приёмов (admission)

Параметры `client_id` / `patient_id` как query-param **не фильтруют** — возвращают все записи.  
Правильный способ: универсальный JSON-параметр `filter`:

```
GET /rest/api/admission?filter=[{"property":"client_id","value":"422"}]&sort=[{"property":"admission_date","direction":"DESC"}]&limit=5
```

Проверено на реальном API: `totalCount` корректно снижается до записей только этого клиента/питомца.

### Счета (invoice)

- Фильтр по `client_id` через `filter` JSON работает аналогично.
- Поле `invoiceDocuments` включено прямо в ответ `GET /rest/api/invoice/{id}` и `GET /rest/api/invoice?...`.
- Каждый `invoiceDocument` содержит `good_id`, `quantity`, `price`, `goodSaleParam`.
- Название товара находится в `goodSaleParam` → нужен отдельный запрос `GET /rest/api/good/{id}` для получения `title`. **Решение:** возвращать `good_id` + `price` + `quantity` без лишних запросов (названия — опционально через `get_good_by_id`).
- Статус оплаты: поле `payment_status` (`full`, `partial`, `none`) и `paid_amount` прямо в счёте.

### Вакцинации (VaccinationCard)

**Эндпоинт:** `GET /rest/api/MedicalCards/Vaccinations?pet_id={id}`

Ключевые поля из реального ответа API:
- `date` — дата вакцинации (`YYYY-MM-DD HH:MM:SS`)
- `date_nexttime` — дата следующей ревакцинации (`YYYY-MM-DD`, может быть пустой строкой)
- `name` — название вакцины
- `medcard_id` — связанная медкарта

Ответ в ключе `data.medicalcards` (не `data.vaccinations`!).

### Подзадача 15.2 — `get_client_profile` в `tools/client.py`

**Оценка:** ~1.5 часа / ~100 строк

Агрегирует данные из 4 запросов:
1. `GET /rest/api/client/{client_id}` — полные данные клиента
2. `GET /rest/api/invoice?filter=[{"property":"client_id","value":"{id}"}]&sort=[{"property":"id","direction":"DESC"}]&limit=5` — последние 5 счетов
3. `GET /rest/api/admission?filter=[{"property":"client_id","value":"{id}"}]&sort=[{"property":"admission_date","direction":"DESC"}]&limit=5` — последние 5 приёмов
4. `GET /rest/api/admission?filter=[{"property":"client_id","value":"{id}"},{"property":"status","value":"active"}]&sort=[{"property":"admission_date","direction":"ASC"}]&limit=1` — следующий приём

Возвращает dict:
```json
{
  "client": {...},
  "last_invoices": [...],
  "last_admissions": [...],
  "next_admission": {...} | null
}
```

### Подзадача 15.3 — `get_pet_profile` в `tools/pet.py`

**Оценка:** ~1 час / ~80 строк

Агрегирует данные из 3 запросов:
1. `GET /rest/api/pet/{pet_id}` — данные питомца
2. `GET /rest/api/medicalcard?filter=[{"property":"patient_id","value":"{id}"}]&sort=[{"property":"id","direction":"DESC"}]&limit=5` — последние 5 медкарт
3. `GET /rest/api/MedicalCards/Vaccinations?pet_id={id}` — все вакцинации

Вычисляет:
- `last_vaccination_date` — `date` из последней записи по `date`
- `next_vaccination_date` — `date_nexttime` из последней вакцинации (если непустая)

Возвращает dict:
```json
{
  "pet": {...},
  "last_medical_cards": [...],
  "vaccinations": [...],
  "last_vaccination_date": "YYYY-MM-DD" | null,
  "next_vaccination_date": "YYYY-MM-DD" | null
}
```

# Этап 64. Ревью визуала

Цель: оценить и улучшить UI лендинга, форм регистрации/логина, дашборда аккаунта.

---

## 64.1 Визуальный аудит лендинга

**Общее впечатление:** Лендинг профессиональный, чистый, с хорошей типографикой и цветовой палитрой (warm earth tones). Структура логичная: hero → capabilities → audience → steps → tech block → FAQ → CTA → footer.

### Проблемы

| # | Severity | Проблема | Описание |
|---|---|---|---|
| L1 | MEDIUM | Hero heading уровень h2, не h1 | Главный заголовок "Данные клиники и команды..." — `<h2>`, при этом первый `<h1>` отсутствует на странице. Нарушение heading hierarchy (SEO + a11y). |
| L2 | LOW | Год в футере "2025" | `© 2025 Vetmanager MCP` — устарел, сейчас 2026. |
| L3 | LOW | Смешение языков | "Vetmanager integration", "Bearer-токены", "MCP runtime" — англицизмы в русском тексте. Приемлемо для технической аудитории, но описание на регистрации "account сервиса" звучит неестественно. |
| L4 | MEDIUM | Hero текст слишком длинный | 3 абзаца текста в hero section до CTA-кнопок — можно потерять пользователя. Рекомендация: оставить 1 абзац, остальное ниже. |
| L5 | LOW | FAQ не раскрывается | Элементы FAQ выглядят как аккордион, но нет видимых индикаторов раскрытия (стрелки/+). |

## 64.2 Визуальный аудит форм

### Регистрация и Логин

**Хорошо:** Чистый layout, центрированная карточка, понятные labels, заметная CTA-кнопка, навигационные ссылки внизу.

| # | Severity | Проблема | Описание |
|---|---|---|---|
| F1 | HIGH | Нет требований к паролю | Форма регистрации не показывает требования (мин. длина, символы). Пользователь узнаёт об ошибке только после submit. |
| F2 | MEDIUM | Нет визуального feedback при ошибке ввода | Поля не подсвечиваются красным, нет inline validation. |
| F3 | MEDIUM | Password не маскируется с toggle | Нет кнопки "показать/скрыть пароль". |
| F4 | LOW | Описание на регистрации смешивает языки | "Создайте account сервиса, который позже получит Vetmanager integration и Bearer-токены" — "account" и "integration" не переведены. |

### Форма интеграции (дашборд)

| # | Severity | Проблема | Описание |
|---|---|---|---|
| F5 | HIGH | Заголовки секций на английском | "Vetmanager integration", "Bearer token issuance", "Current tokens" — на русскоязычном дашборде. Inconsistent с "Личный кабинет". |
| F6 | MEDIUM | Placeholder "myclinic" в domain field | Не очевидно что нужно вводить. Нужен hint: "поддомен клиники, например: myclinic" с пояснением что это часть URL myclinic.vetmanager.cloud. |
| F7 | MEDIUM | "Переавторизоваться и обновить токен" — длинная кнопка | Текст не помещается на мобильных viewport-ах, кнопка переносится. |
| F8 | LOW | Placeholder "Cursor production" для token name | Подсказка может ввести в заблуждение — не все используют Cursor. |

## 64.3 Визуальный аудит дашборда

**Хорошо:** Информативные status cards (Статус аккаунта / Активные интеграции / Bearer-токены), логичный flow сверху вниз (info → integration → tokens), privacy transparency block.

| # | Severity | Проблема | Описание |
|---|---|---|---|
| D1 | HIGH | Нет визуального разделения секций | Integration form, token issuance, token list — визуально сливаются. Нет separators/cards для секционирования. |
| D2 | MEDIUM | Privacy block занимает много места | Два абзаца о privacy и auth transparency — полезно при первом визите, но мешает при повторных. Рассмотреть collapsible. |
| D3 | MEDIUM | "Токенов пока нет." — одинокое сообщение | Empty state для токенов не информативен. Добавить подсказку: "Выпустите первый токен после настройки интеграции". |
| D4 | LOW | Кнопка "Выйти" в самом низу | На длинной странице неудобно искать logout. Добавить в header или top section. |

## 64.4 Проверка доступности (a11y)

| # | Severity | Проблема | Описание |
|---|---|---|---|
| A1 | HIGH | Отсутствует h1 на лендинге | Heading hierarchy начинается с h2. Screen reader не найдёт главный заголовок. |
| A2 | MEDIUM | Нет `<main>` landmark на лендинге | Content вложен в `<div>`, не в `<main>`. Формы регистрации/логина имеют `<main>`. |
| A3 | MEDIUM | Нет aria-label на nav | Navigation landmark без label — screen reader не различит header nav от footer nav. |
| A4 | MEDIUM | FAQ details/summary без aria | Аккордион не имеет aria-expanded/aria-controls для assistive tech. |
| A5 | LOW | Form fields без autocomplete | Email field без `autocomplete="email"`, password без `autocomplete="current-password"` / `autocomplete="new-password"`. |
| A6 | LOW | Нет skip-to-content link | Keyboard navigation: нельзя быстро перейти к основному контенту. |

## 64.5 Мобильная адаптивность

**375px (iPhone SE) viewport протестирован.**

| # | Severity | Проблема | Описание |
|---|---|---|---|
| M1 | HIGH | Лендинг header: логотип и навигация обрезаются | На 375px header elements collapse неудачно. Нет hamburger menu. Ссылки навигации мелкие и трудно нажимаемы. |
| M2 | MEDIUM | Hero text мелкий и плотный | Длинные абзацы в hero section тяжело читать на мобильном. |
| M3 | MEDIUM | Feature cards не stack правильно | Cards "Клиенты и пациенты", "Приёмы, финансы" — текст сжимается. |
| M4 | MEDIUM | Дашборд: radio buttons для auth mode тесные | "Подключить по API key" / "Подключить по логину и паролю" — текст переносится неудачно. |
| M5 | LOW | CTA кнопки в footer лендинга — "Создать аккаунт" и "Войти" stack вертикально | Визуально приемлемо, но кнопки мелкие для touch target (< 44px height). |

## 64.6 Сводный план исправлений

### Приоритет 1 (HIGH):
1. **A1/L1** — Исправить heading hierarchy: hero heading → h1
2. **F1** — Добавить требования к паролю в форму регистрации (видимые до submit)
3. **F5** — Перевести заголовки дашборда на русский
4. **D1** — Визуально разделить секции дашборда (cards/separators)
5. **M1** — Добавить hamburger menu для мобильного header

### Приоритет 2 (MEDIUM):
6. **F2** — Inline validation / подсветка ошибок полей
7. **F3** — Toggle показа/скрытия пароля
8. **F6** — Улучшить hint для domain field
9. **F7** — Сократить текст кнопки "Переавторизоваться"
10. **D2** — Сделать privacy block collapsible
11. **D3** — Улучшить empty state для токенов
12. **A2–A4** — Добавить landmark roles и aria атрибуты
13. **L4** — Сократить hero text
14. **M2–M4** — Улучшить мобильную адаптивность текста и форм

### Приоритет 3 (LOW):
15. **L2** — Обновить год в футере
16. **L3/F4** — Унифицировать язык (перевести англицизмы или оставить осознанно)
17. **A5–A6** — autocomplete атрибуты, skip-to-content
18. **D4** — Logout в header
19. **M5** — Увеличить touch targets кнопок

## 64.7 Вывод

UI функционален и визуально приятен, но имеет проблемы с:
- **a11y** (heading hierarchy, landmarks, aria)
- **мобильной адаптивностью** (header navigation, плотный текст)
- **language consistency** (EN заголовки на RU дашборде)
- **form UX** (нет password requirements, inline validation)

Критических layout/rendering багов нет. Реализация улучшений — в этапе 69.

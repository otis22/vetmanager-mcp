# Этап 129.8. Shared save_connection helper

## Контекст

В `vetmanager_connection_service.py` три save-path (`domain_api_key`, `user_token`, `login_password`)
повторяют общий lock/transaction/disable-old/create/set_credentials/refresh flow.

## Цель

Свести повторяющийся persistence path к одному `_save_connection(...)` helper без изменения
validation semantics и dedupe behavior из stage `129.3`.

## Scope

1. Добавить `_save_connection(...)`.
2. Перевести три save entrypoints на него.
3. Сохранить reuse-existing logic для login/password path.
4. Прогнать существующий connection-service regression contour.

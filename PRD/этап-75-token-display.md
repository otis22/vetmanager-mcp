# PRD: Этап 75 — Улучшение отображения нового bearer-токена

## Цель

При выпуске нового bearer-токена показать его максимально явно и удобно для копирования.

## Проблема

Текущая реализация: raw token отображается в input[readonly] внутри token-flash секции. Токен длинный и обрезается, не видно полностью. Предупреждение есть, но не достаточно акцентировано.

## Требования

1. **Полный токен** виден целиком — не обрезается, word-break для переноса
2. **Кнопка "Скопировать"** — явная, заметная
3. **Предупреждение** — "Токен показывается только один раз. После перезагрузки страницы он будет недоступен." — выделено визуально (warning-стиль)
4. **Инструкция подключения** — пример JSON для mcp.json с подставленным токеном

## Декомпозиция

### 75.1 Переработать token-flash (≤1.5 ч, ≤80 строк)

Заменить текущий token-flash HTML:
- Убрать input, показать токен в `<code>` блоке с word-break
- Кнопка копирования рядом
- Предупреждение в стиле warning (акцентный цвет, иконка)
- Блок с примером подключения (JSON с подставленным токеном)

### 75.2 Обновить тесты (≤30 мин)

Адаптировать `test_account_token_issue_shows_raw_token_once_and_stores_only_hash`

в `web_security.py` ещё остаётся отдельный public helper `record_rate_limit_hit(...)`, который
поддерживает старую split API model.

## Цель

append helper и переведя тесты/вызовы на consume-based priming.

2. Перевести тесты, которые вручную набивают лимитер, на `consume_rate_limit(...)`.

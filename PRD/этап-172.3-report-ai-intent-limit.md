# Этап 172.3. Report AI `intent_text` limit

## Контекст

Production feedback `#8`: текущий лимит `intent_text=1000` мешает формулировать сложные аналитические запросы для Report AI, например ABC/XYZ и многоусловные отчёты. Roadmap-цель этапа — поднять лимит до 64 000 символов, если это поддерживает upstream; иначе синхронизировать MCP с максимально подтверждённым значением.

## Проверенные факты

- MCP сейчас валидирует `tools/report_ai.py::INTENT_MAX_LENGTH = 1000`.
- Upstream `JobService::INTENT_MAX_LENGTH = 1000` валидирует `mb_strlen($intent) > 1000` и возвращает `VALIDATION_ERROR`.
- Upstream `ReportAiJobsController::actionCreate()` не задаёт отдельный max length, но передаёт `intent_text` в `JobService::create()`.
- Upstream storage `report_ai_jobs.intent_text` — `TEXT NOT NULL`, то есть storage не является непосредственным блокером для 64 000 символов.
- Real API probe на `devtr6` через проектный `VetmanagerClient` подтвердил: `POST /rest/api/report-ai-job` с `intent_text` длиной 1001 возвращает HTTP 400 `VALIDATION_ERROR — intent_text длиннее 1000 символов`.

## Решение

Не поднимать MCP `INTENT_MAX_LENGTH` выше 1000 в этом этапе.

Причина: MCP-only изменение ухудшит контракт. Tool перестанет давать быстрый понятный client-side reject, но upstream всё равно отвергнет запрос с тем же лимитом. Это не разблокирует пользователя и добавит лишний сетевой round-trip.

## Scope

### In scope

- Зафиксировать upstream blocker в PRD, Roadmap и AssumptionLog.
- Оставить MCP validation/docs/tests в текущем подтверждённом состоянии.
- Перейти к следующей задаче Stage 172.

### Out of scope

- Изменение `vetmanager-extjs` upstream-кода.
- Миграция storage или deploy upstream.
- Создание MCP обхода через разбиение prompt: Report AI endpoint принимает один `intent_text`, а не multipart-инструкции.

## Acceptance

- Максимально подтверждённое значение для текущего production/devtr6 контура зафиксировано как `1000`.
- MCP не принимает значения, которые текущий upstream гарантированно отвергает.
- Roadmap item `172.3` переведён в `stop` с причиной upstream blocker.

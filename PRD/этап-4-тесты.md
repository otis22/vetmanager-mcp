# PRD Этап 4: Тестирование

## Цель
Обеспечить минимум 20 e2e-сценариев: mock/contract по OpenAPI и real API тесты для домена devtr6.

## Задачи

### 4.1 Unit-тесты (уже реализованы в Этапе 1)
- 6 unit-тестов в tests/test_client_multitenancy.py

### 4.2 E2E mock/contract тесты
- Проверка всех инструментов через respx-моки
- Минимум 12 сценариев
- Ограничение: ≤ 200 строк

### 4.3 E2E real API тесты
- Запускаются только если TEST_DOMAIN и TEST_API_KEY заданы
- Минимум 8 сценариев через реальный API devtr6
- Ограничение: ≤ 150 строк

### 4.4 Итого: ≥ 20 сценариев (6 unit + 12 mock + 8 real ≥ 26)

## Критерии готовности
- `docker compose run --rm test pytest tests/ -v` проходит все тесты
- Real API тесты пропускаются (skip) если нет TEST_API_KEY

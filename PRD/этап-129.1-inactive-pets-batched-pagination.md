# PRD: Этап 129.1 — inactive-pets batched pagination

## Контекст

Post-review 2026-04-21 нашёл silent truncation в batched inactive-pets path:
- invoice/medcard chunk helpers используют `limit=100, offset=0`;
- если один `IN` chunk возвращает больше 100 rows из-за нескольких invoices/medcards на pet/day, хвост silently теряется;
- итоговый `get_inactive_pets` может ложно исключать животных, которые присутствуют только на следующих страницах chunk response.

## Цель

Сделать batched invoice/medcard lookup fully paginated для обоих путей:
- `find_pets_at_client_last_visit(...)`
- `find_pets_for_clients_last_visit(...)`

## Scope

В scope:
- `tools/_inactive_helpers.py`
- `tests/test_inactive_pets.py`
- `Roadmap.md`, `AssumptionLog.md`

Вне scope:
- fan-out limiting
- generic refactor helper extraction beyond minimal reusable pagination helper

- regression test на `>100` medcard rows в одном chunk

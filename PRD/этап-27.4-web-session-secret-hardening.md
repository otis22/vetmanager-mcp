# PRD: Этап 27.4. Security hardening web-сессий и secret management

## Цель

Усилить безопасность web session и управления секретами без ломки текущего
account-based web контура.

## Выявленные слабые места

- `web_auth.get_web_session_secret()` сейчас допускает hardcoded fallback
  `dev-web-session-secret`.
- Cookie web-сессии выставляется с:
  - `HttpOnly=true`
  - `SameSite=lax`
  - `Secure` только по env-флагу, по умолчанию фактически `False`.
- Такой контракт слишком мягкий для production-like bearer service.

## Граница текущего решения

- Не добавляется полноценный CSRF subsystem.
- Не добавляется server-side session storage.
- Не меняется cookie-based модель сессии.

## Что усиливаем на этом шаге

- Убираем hardcoded dev session secret.
- Секрет подписи web session должен приходить из:
  - `WEB_SESSION_SECRET`, либо
  - fallback в `STORAGE_ENCRYPTION_KEY`.
- Cookie defaults:
  - `HttpOnly=true` остаётся;
  - `SameSite=strict` по умолчанию;
  - `Secure=true` по умолчанию;
  - для локальных тестов и dev допускается явный env override.

## Почему допустим fallback в `STORAGE_ENCRYPTION_KEY`

- Это лучше, чем встроенный слабый dev secret.
- Не требует немедленной миграции всей текущей локальной конфигурации.
- При этом явный `WEB_SESSION_SECRET` остаётся предпочтительным production-вариантом.

## Декомпозиция

### 27.4.1 Session secret policy
- Убрать hardcoded fallback secret.
- Возвращать явную ошибку при отсутствии и `WEB_SESSION_SECRET`, и `STORAGE_ENCRYPTION_KEY`.

### 27.4.2 Cookie hardening
- Добавить helper вычисления cookie security settings.
- Сделать `Secure=true` и `SameSite=strict` дефолтами.
- Оставить env override для тестового/локального контура.

### 27.4.3 Tests
- Unit test: без session secret конфигурации helper падает.
- Unit test: cookie defaults становятся secure/strict.
- Web tests продолжают работать через явный test override.

## Критерии готовности

- В коде больше нет hardcoded dev session secret.
- Web session cookie по умолчанию имеет более строгие security attributes.
- Тестовый контур остаётся управляемым через env override.

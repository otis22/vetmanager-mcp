# PRD Этап 6: Операционные скрипты init/deploy по SSH

## Цель
Два bash-скрипта для управления сервером на удалённой машине по SSH.
Предусловие: `ssh-copy-id user@host` уже выполнен.
Всё выполняется через Docker на удалённой машине — Python на хосте не нужен.

## Задачи

### 6.1 scripts/init_server.sh
- Подключается по `ssh user@host`
- Устанавливает Docker если не установлен
- Клонирует/копирует репозиторий
- Создаёт `.env` на сервере (через интерактивный ввод или аргументы)
- Запускает `docker compose build`
- Ограничение: ≤ 80 строк

### 6.2 scripts/deploy_server.sh
- Подключается по `ssh user@host`
- git pull (обновляет код)
- `docker compose build` (пересобирает образ)
- `docker compose down && docker compose up -d` (перезапуск)
- Smoke-проверка что контейнер поднялся
- Ограничение: ≤ 60 строк

### 6.3 Smoke-проверка и инструкция
- После деплоя проверить `docker compose ps`
- Задокументировать использование в README

## Критерии готовности
- `scripts/init_server.sh user@host` — полностью настраивает сервер
- `scripts/deploy_server.sh user@host` — обновляет код и перезапускает
- Никакого Python на хосте или целевом сервере кроме Docker

# Этап 129.4. Bounded concurrency for inactive pets batched lookup

## Контекст

После перехода `get_inactive_pets` на batched owner/day lookup в `tools/_inactive_helpers.py`
появилось несколько `asyncio.gather(...)` без ограничения fan-out. На крупных страницах клиентов
это может создавать burst на десятки одновременных upstream GET-запросов к Vetmanager API.

## Проблема

- `find_pets_at_client_last_visit(...)` параллелит invoice и medcard chunk lookup без concurrency cap.
- `find_pets_for_clients_last_visit(...)` параллелит pet, invoice и medcard chunk lookup без concurrency cap.
- Поведение функционально корректно, но под нагрузкой создаёт избыточный pressure на upstream и ухудшает predictability.

## Цель

Ввести bounded concurrency внутри `_inactive_helpers.py`, не меняя внешний контракт `get_inactive_pets`
и не ломая существующую batched/paginated логику.

## Scope

1. Добавить локальный helper для bounded gather с сохранением порядка результатов и exception semantics.
2. Перевести все `asyncio.gather(...)` в `_inactive_helpers.py`, относящиеся к batched owner/day lookup, на bounded helper.
3. Добавить regression tests, подтверждающие, что:
   - single-owner path (`find_pets_at_client_last_visit`) не превышает cap при invoice chunk lookup;
   - multi-client path (`find_pets_for_clients_last_visit`) не превышает cap при batched owner/day lookup.

## Не входит

- Глобальная конфигурация concurrency cap через env.
- Рефакторинг повторяющейся day-window logic между invoice и medcard path.
- Изменение бизнес-логики отбора inactive pets.

## Декомпозиция

1. Добавить constants/helper для bounded concurrency в `tools/_inactive_helpers.py`.
2. Перевести существующие chunk gathers на bounded helper.
3. Добавить unit-level regression tests в `tests/test_inactive_helpers.py`.
4. Прогнать targeted tests и полный suite.

## Риски

- Слишком низкий cap может ухудшить latency на больших batch pages.
- Неправильная реализация bounded helper может изменить порядок результатов или скрыть exception propagation.

## Критерии готовности

- Все batched gathers в `_inactive_helpers.py` проходят через bounded helper.
- Тесты подтверждают `max_in_flight <= cap`.
- Regression suite зелёный.

# Этап 172.5. `get_average_invoice` date basis

## Контекст

Production feedback `#6`: за день, где `get_revenue_summary` и admissions показывают выручку/активность, `get_average_invoice` возвращает нули. Текущий `get_average_invoice` фильтрует `invoice.create_date`, тогда как финансовая дата выполненного счёта в проекте уже закреплена как `invoice.invoice_date`.

## Проверенные факты

- Stage 144 зафиксировал: `invoice.create_date` — дата создания записи, `invoice.invoice_date` — финансовая дата проведённого счёта.
- `get_revenue_summary(mode="invoiced")` и `mode="paid_by_executed_invoices"` используют `invoice_date` + `status="exec"` + half-open day boundary.
- `get_invoices(invoice_date_from/to)` уже использует `invoice_date >= day_start` и `invoice_date < next_day_start` и добавляет `status="exec"` при отсутствии явного статуса.
- `get_average_invoice` сейчас использует `create_date >= date_from` и `create_date <= date_to`, не возвращает `date_field` и не сообщает семантику даты.

## Цель

Сделать семантику даты явной и привести default к финансовому сценарию morning brief: средний чек за день должен считаться по выполненным счетам за `invoice_date`.

## Scope

### In scope

- Добавить аргумент `date_basis`:
  - `invoice_date` — default, financial/executed invoice date;
  - `create_date` — explicit audit/record-created fallback.
- Для обоих basis использовать half-open day window.
- Для `invoice_date` учитывать только `status="exec"`.
- Для `create_date` сохранить legacy fallback: не добавлять status filter автоматически.
- Вернуть в output `date_basis`, `date_field`, `amount_field`, `status`, `applied_filters`, `warnings`.
- Сохранить совместимые поля `invoices_with_amount`, `total_revenue`, `average_invoice`.
- Покрыть default `invoice_date`, explicit `create_date` и invalid `date_basis` тестами.

### Out of scope

- Payment-based average check.
- Смена `get_revenue_summary`.
- Doctor/clinic/group breakdowns.

## Acceptance

- `get_average_invoice(date_from="2026-06-17", date_to="2026-06-17")` фильтрует `invoice_date >= 2026-06-17 00:00:00`, `invoice_date < 2026-06-18 00:00:00`, `status="exec"`.
- `get_average_invoice(..., date_basis="create_date")` фильтрует `create_date` тем же half-open window, не добавляет status filter и явно возвращает warning про legacy/audit semantics.
- Invalid `date_basis` rejected before HTTP.
- Targeted Docker tests pass.

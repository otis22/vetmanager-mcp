# PRD: Этап 42.7. Cleanup helper для browser tests

## Контекст

Browser happy-path tests уже создают реальные account/integration/token сущности
в тестовой БД. Следующий шаг — сделать явный cleanup helper, чтобы browser tests
удаляли тестовый account и все связанные записи предсказуемо и единообразно.

## Цель 42.7

Добавить reusable helper/fixture для browser tests, который:
- регистрирует test account по email;
- выполняет cleanup по требованию и в teardown;
- удаляет связанные Vetmanager connections, bearer tokens, usage stats и logs.

## Входит

- Новый helper/fixture в test infrastructure.
- Подключение helper к существующим browser happy-path tests.
- Подготовка базы для отдельного regression test cleanup.

## Не входит

- Optional real browser mode.
- README updates.

## Критерии готовности

- Browser tests используют единый cleanup helper.
- Cleanup можно вызвать вручную внутри теста и он безопасен в финальном teardown.
- Helper даёт прозрачный отчёт по counts до/после очистки.

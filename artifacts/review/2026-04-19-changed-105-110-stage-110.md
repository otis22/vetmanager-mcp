## Blockers (detail)

### B1. Business events counter на публичном `/metrics`
**Files:** `web_routes_system.py:47-53` (`/metrics` endpoint), `service_metrics.py:349-362` (render), `scripts/init_server.sh:149-158` (nginx proxy без allow/deny).
**Problem:** stage 110 добавил `vetmanager_business_events_total{event="..."}` в публичный `/metrics`. stage 107 добавил `auth_failures_total{source="web_register",reason="rate_limited"}` туда же. Nginx проксирует `/` без IP-allowlist → любой интернет-пользователь может скрейпить `https://vetmanager-mcp.vromanichev.ru/metrics` каждые 15s и собирать:
- cadence регистраций (product-sensitive)
- velocity выпуска/отзыва токенов (churn proxy)
- login-success rate (timing credential-stuffing — можно подгадывать burst'ы под лимиты)

**Why it matters:** Конкурент получает free growth dashboard; атакующий использует timing для уклонения от лимитов. Ни оператор, ни alerting не заметят scraping.
**Fix:** `location = /metrics { allow 127.0.0.1; allow <monitoring-ip>; deny all; }` в nginx **перед** `location /`. Defence-in-depth: `METRICS_AUTH_TOKEN` env check в `metrics_export`. Либо вынести business events в отдельный `CollectorRegistry`, не mount'ить на публичный endpoint.
**Confirmed by:** security(high, 0.85), observability(косвенно через business_events coverage), codex-blindspot(косвенно).

### B2. Inline imports после "cleanup" stage 108 (3 high)
**Files:**
- `service_metrics.py:143` — `import time as _time` в `instrument_call` (hot path)
- `resources/_aggregation.py:67-71, 96` — `from exceptions import (...)` inline + дубль `from exceptions import AuthError` отдельной строкой
- `vm_transport/breaker.py:135` — `from observability_logging import RUNTIME_LOGGER` в `breaker_record_success`

**Problem:** Stage 108 явно заявлял "inline imports" одной из целей. Три из четырёх inline импортов **не циркулярны** — просто забыты либо defensively inline'нуты. Reviewer-code помечает все три с confidence 0.90-0.95.

**Why it matters:** dict-lookup на каждый вызов (cost есть, хоть и мал); прячет зависимость из header; в случае `resources/_aggregation.py:96` — пример того, что stage 108 прошёл мимо простого дубля.
**Fix:** переместить на module level; удалить строку 96 в `_aggregation.py`. ~15 LOC diff, 0 риска.
**Confirmed by:** code(high×3 findings).

### B3. Missing composite index `token_usage_logs(event_type, event_at)`
**File:** `alembic/versions/20260321_000001_bearer_service_baseline.py:62-72` (нет индекса); consumer: `scripts/product_metrics_report.py:293-347` (10 sequential queries).
**Problem:** `scripts/product_metrics_report.py` делает 7×`_count_events` + 3×`_failure_breakdown` — все `WHERE event_type = X AND event_at >= Y` (+ GROUP BY для breakdown). Без composite index — full table scans. Наполнение TokenUsageLog растёт как ~1000 req/min × 60 × 24 = **~1.4M строк/сутки** на busy tenant base. При 100k+ строк CLI будет > 10s; при 1M+ — десятки секунд и throw-off concurrent writes (SQLite WAL serialization, Postgres buffer cache eviction).
**Why it matters:** stage 110 PRD обещает <2s на prod. Сейчас работает только потому что база юная.
**Fix:** alembic миграция:
```python
op.create_index("ix_token_usage_logs_event_type_event_at", "token_usage_logs", ["event_type", "event_at"])
```
Плюс параллельная батчированная единая CTE-query вместо 10 последовательных.
**Confirmed by:** performance-and-reliability(high, 0.95).

### B4. `record_business_event` silent-drop design
**File:** `service_metrics.py:72-86`.
**Problem:** `_ALLOWED_BUSINESS_EVENTS` — frozenset из 4 literal event_name. Unknown event → `return` без лога. Typo в новом call-site (например `account_register` вместо `account_registered`) молча инкрементирует nothing; operator смотрит Grafana, видит 0, расследует несуществующий incident.
**Why it matters:** реальный debug trap, на который уже указали 4 ревьюера (simplicity, architecture, observability, code). Stage 110 PRD позиционирует counter как hook для будущего Grafana — hard-coded allowlist инвертирует это обещание.
**Fix:** (a) `RUNTIME_LOGGER.warning("record_business_event: unknown event_name dropped", extra={"dropped_name": event_name})` перед `return` **или** (b) `BusinessEvent` Enum так что typo fail at call-site **или** (c) удалить allowlist (все 4 callsite literal, cardinality не взорвётся). Рекомендуется (a)+(b).
**Confirmed by:** observability(medium, 0.92), architecture(medium, 0.55), simplicity(low, 0.55), code(косвенно).

# Справочник сущностей Vetmanager API — Доменные имена и синонимы

---

## 1. Введение

Этот документ является расширенной версией справочника сущностей Vetmanager REST API (v6), дополненной **доменными именами и синонимами** для нужд RAG-сервиса. Каждая сущность и её ключевые поля снабжены перечнем альтернативных наименований, которые реально используются в базе знаний, пользовательских запросах и интерфейсе программы.

**Назначение документа:** улучшить качество семантического поиска — когда пользователь спрашивает «как найти хозяина кота» или «покажи приходную накладную», RAG должен корректно сопоставить эти запросы с сущностями `client`/`pet` и `storeDocument` соответственно.

### 2.39. Messages (Уведомления пользователям)

> **Бизнес-контекст:** API для отправки уведомлений (сообщений) пользователям программы. Текст сообщения поддерживает HTML.

**Эндпоинт API:**
- `POST /rest/api/messages/all`
- `POST /rest/api/messages/users`
- `GET /rest/api/messages/reports?campaign=...`
- `POST /rest/api/messages/roles`

Тело POST: `message` (строка, HTML), `campaign` (название кампании); для `users` — дополнительно `user_ids` (массив ID пользователей); для `roles` — дополнительно `roles` (массив названий ролей, например `["Врач"]`). GET reports — query-параметр `campaign`.

**Доменные синонимы сущности:**
`уведомление`, `рассылка`, `сообщение в программу`, `уведомление пользователям`, `push в программу`, `messages`

| Метод | Назначение | Ключевые параметры | Доменные формулировки |
| :--- | :--- | :--- | :--- |
| `POST /rest/api/messages/all` | Массовая рассылка всем пользователям | `message`, `campaign` | отправить всем, общая рассылка, массовое уведомление |
| `POST /rest/api/messages/users` | Адресная рассылка конкретным пользователям | `message`, `campaign`, `user_ids` | написать сотруднику, отправить выбранным людям, адресное уведомление |
| `GET /rest/api/messages/reports` | Получить отчёт по кампании | `campaign` (фактически обязателен), `limit`, `offset`, `sort`, `filter` | статус рассылки, отчёт по уведомлению, сколько отправлено |
| `POST /rest/api/messages/roles` | Рассылка по ролям | `message`, `campaign`, `roles` | отправить врачам, уведомить администраторов, рассылка по должности |

> **Примечание:** В реальном API для `GET /rest/api/messages/reports` query-параметр `campaign` фактически обязателен, хотя в OpenAPI это не отражено явно.

**Примеры payload'ов:**

`POST /rest/api/messages/all`
```json
{
  "message": "Rest post",
  "campaign": "All1"
}
```

`POST /rest/api/messages/users`
```json
{
  "message": "Rest post",
  "campaign": "Concrete1",
  "user_ids": [1]
}
```

`POST /rest/api/messages/roles`
```json
{
  "message": "Rest post",
  "campaign": "Concrete1",
  "roles": ["Врач"]
}
```

**Пример ответа для reports:**
```json
{
  "success": true,
  "data": {
    "campaign": "All users",
    "total": 0,
    "sent": 0,
    "pending": 0
  }
}
```

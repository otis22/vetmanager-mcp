
# Справочник сущностей Vetmanager API

**Автор:** Manus AI
**Версия:** 1.0
**Дата:** 27 февраля 2026 г.

---

## 1. Введение

Этот документ представляет собой подробный справочник по сущностям данных, доступным через Vetmanager REST API (v6). Для каждой сущности объясняется ее роль в системе управления ветеринарной клиникой Vetmanager, указывается эндпоинт API и описывается каждое поле в ее схеме. Этот справочник предназначен служить основной базой знаний для разработчиков, создающих интеграции, включая Vetmanager MCP Server.

Информация синтезирована из официальной спецификации Vetmanager OpenAPI v6 [1], портала помощи Vetmanager [2] и коллекции Postman [3].

## 2. Справочник сущностей

### 2.1. Admission (Приём)

> **Бизнес-контекст:** Сущность `admission` (приём) представляет собой запись о визите в расписании клиники. Это ядро рабочего процесса, связывающее клиента, его питомца и ветеринара в определенную дату и время.

**Эндпоинт API:** `/rest/api/admission`

| Имя поля | Тип | Nullable | Описание |
| :--- | :--- | :--- | :--- |
| `id` | integer | Нет | Уникальный идентификатор приёма. |
| `admission_date` | string | Нет | Дата и время приёма. Формат: `YYYY-MM-DD HH:MM:SS`. |
| `description` | string | Нет | Описание или примечание к приёму. |
| `client_id` | integer | Нет | Внешний ключ, связывающий с сущностью `client` (владелец животного). |
| `patient_id` | integer | Нет | Внешний ключ, связывающий с сущностью `pet` (питомец). |
| `user_id` | integer | Нет | Внешний ключ, связывающий с сущностью `user` (ветеринар или сотрудник). |
| `type_id` | integer | Нет | Тип приёма (например, первичный, повторный, вакцинация). Ссылается на элемент справочника. |
| `admission_length` | string | Нет | Продолжительность приёма (например, `00:30:00`). |
| `status` | string | Нет | Текущий статус приёма (например, `active`, `deleted`, `completed`). |
| `clinic_id` | integer | Нет | Внешний ключ, связывающий с сущностью `clinics`. Актуально для сетей клиник. |
| `direct_direction` | integer | Нет | Указывает направление потока приёма. |
| `creator_id` | integer | Нет | ID пользователя, создавшего запись о приёме. |
| `create_date` | string | Нет | Временная метка создания записи. Формат: `YYYY-MM-DD HH:MM:SS`. |
| `escorter_id` | integer | Нет | ID сопровождающего сотрудника, если применимо. |
| `reception_write_channel` | string | Нет | Канал, через который был забронирован приём (например, `phone`, `website`, `walk-in`). |
| `is_auto_create` | integer | Нет | Флаг, указывающий, был ли приём создан автоматически системой. |
| `invoices_sum` | string | Нет | Общая сумма всех счетов, связанных с этим приёмом. |
| `confirmation` | string | Нет | Статус подтверждения приёма. |
| `pet` | object | Да | Вложенный объект с данными питомца. |
| `client` | object | Да | Вложенный объект с данными клиента. |
| `wait_time` | string | Нет | Расчетное время ожидания для клиента. |
| `invoices` | array | Нет | Массив объектов счетов, связанных с этим приёмом. |

### 2.36. VaccinationCard (Запись о вакцинации)

> **Бизнес-контекст:** Сущность `VaccinationCard` представляет отдельную запись о введённой вакцине из медицинской карты питомца. Доступна через специальный эндпоинт `MedicalCards/Vaccinations`, который принимает фильтр `pet_id`. Поля `date` и `date_nexttime` позволяют отследить дату последней вакцинации и дату следующей ревакцинации.

**Эндпоинт API:** `GET /rest/api/MedicalCards/Vaccinations?pet_id={id}`

> **Примечание:** Эндпоинт не является стандартным REST-ресурсом и не поддерживает универсальные параметры `filter`/`sort`. Фильтрация по питомцу осуществляется только через query-параметр `pet_id`. Ответ возвращается в ключе `data.medicalcards`.

| Имя поля | Тип | Nullable | Описание |
| :--- | :--- | :--- | :--- |
| `id` | integer | Нет | Уникальный идентификатор записи о вакцинации. |
| `name` | string | Нет | Название вакцины (например, `«Биовак DPAL»`). |
| `pet_id` | integer | Нет | Внешний ключ к сущности `pet`. |
| `date` | string | Нет | Дата проведения вакцинации. Формат: `YYYY-MM-DD HH:MM:SS`. |
| `date_nexttime` | string | Да | Дата следующей ревакцинации. Формат: `YYYY-MM-DD`. Пустая строка, если не задана. |
| `vaccine_id` | integer | Нет | ID препарата-вакцины в справочнике товаров. |
| `birthday` | string | Да | Дата рождения питомца на момент вакцинации (может быть `null`). |
| `birthday_at_time` | string | Нет | Возраст питомца в момент вакцинации (строковое представление). |
| `medcard_id` | integer | Нет | Внешний ключ к медицинской карте (`MedicalCard`), в рамках которой была сделана запись. |
| `doza_type_id` | integer | Нет | ID типа дозировки (ссылается на справочник). |
| `doza_value` | string | Нет | Значение дозы (числовая строка, например `"1.0000000000"`). |
| `sale_param_id` | integer | Нет | ID параметра продажи товара (`GoodSaleParam`). |
| `vaccine_type` | integer | Нет | Тип вакцины (числовой код). |
| `vaccine_description` | string | Нет | Дополнительное описание вакцины. |
| `vaccine_type_title` | string | Нет | Текстовое название типа вакцины. |
| `next_admission_id` | integer | Нет | ID следующего приёма, созданного для ревакцинации (0 если нет). |
| `next_visit_time` | string | Нет | Время следующего визита (пустая строка если не задано). |
| `pet_age_at_time_vaccination` | string | Нет | Возраст питомца на момент вакцинации (строка, например `«не указано»`). |

**Пример ответа (сокращённо):**
```json
{
  "data": {
    "medicalcards": [
      {
        "id": 52,
        "name": "Биовак DPAL",
        "pet_id": 66,
        "date": "2026-03-01 00:00:00",
        "date_nexttime": "2026-04-01",
        "vaccine_id": 260,
        "medcard_id": 800,
        "doza_value": "1.0000000000",
        "next_admission_id": 0
      }
    ]
  },
  "success": true
}
```
